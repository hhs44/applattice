# ADR-0001：分离业务流程编排与 AI Agent 编排

- 状态：建议采纳
- 日期：2026-07-17

## 背景

平台同时包含确定性的端到端业务流程和非确定性的 AI 任务。前者要求版本化、状态一致、定时器、重试、补偿和审计；后者需要模型调用、工具使用、人工审核和上下文记忆。

## 决策

- 使用 Temporal、Camunda 或 Flowable 一类工作流引擎承载业务流程状态。
- 使用 LangGraph 承载 AI Agent 图、RAG、模型调用和人工确认。
- 两类编排通过结构化命令和领域事件协作。
- AI 不能直接提交高风险副作用；必须经过业务流程、权限和审批控制。

## 影响

### 正向

- 业务状态可重放、可补偿、可审计。
- AI 能力可以独立迭代，不污染核心业务一致性。
- 故障和责任边界更清晰。

### 代价

- 需要维护两类编排运行时和契约。
- 需要统一追踪 ID、事件 Schema 和运维视图。

## 被否决方案

- 仅使用 LangGraph 编排所有业务：难以满足确定性、补偿和长期版本治理。
- 仅使用传统工作流引擎实现 AI：模型上下文、工具循环和人工中断表达能力不足。
