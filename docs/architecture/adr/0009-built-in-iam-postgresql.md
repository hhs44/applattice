# ADR-0009：v0.2 提供内建 IAM 并使用 PostgreSQL

- 状态：Accepted
- 日期：2026-07-23
- 修订：ADR-0007 中“首版不提供完整 IAM”的产品边界

## 背景

v0.1 Gateway 只验证企业 OIDC Token，并在开发模式下合成主体。v0.2 需要同时服务没有企业 IdP 的内网环境，以及需要 OIDC 联邦、角色绑定、会话撤销和统一审计的组织。

## 决策

新增独立 Control Plane/IAM 服务。它管理本地账号、OIDC 外部身份关联、组、角色、作用域绑定和服务端不透明会话，数据存储在 PostgreSQL。Gateway 通过内部接口内省会话并执行 API 授权，允许最多 30 秒权限缓存。

AppLattice 不作为第三方系统的 OIDC Provider。v0.2 按单组织、小型单节点设计，但数据库迁移、备份和恢复是正式交付要求。

## 后果

平台可以在纯内网中独立登录，也能联邦企业身份；同时承担密码安全、会话、账号恢复、数据库迁移和审计责任。MFA、双人审批和 WORM 审计延后，不降低密码、Cookie、CSRF 和生产开发模式隔离的基础安全要求。
