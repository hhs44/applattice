# ADR-0010：拆分应用、版本与部署生命周期

- 状态：Accepted
- 日期：2026-07-23

## 背景

“已发布”“已回滚”“已归档”分别描述版本、环境和应用本体。将它们放入一个状态机会导致回滚篡改版本历史、归档误删部署信息，并让多环境发布无法表达。

## 决策

使用三个相互关联的状态机：应用状态表达存续策略，版本状态表达注册与验证，部署状态表达环境修订的运行结果。发布和回滚始终创建新的部署修订；版本不可变；归档要求无活动部署并保留历史。

## 后果

模型更容易审计和回滚，也能支持同一版本在不同环境具有不同状态。代价是 API、界面和测试必须明确状态所属实体，不能再使用含义模糊的单一 `status`。
