# ADR-0003：领域能力采用模块化单体优先

- 状态：建议采纳
- 日期：2026-07-17

## 背景

当前业务边界仍在演进，存量系统需要渐进接入，团队规模和独立扩缩容需求尚未明确。直接按能力拆成大量微服务会提前引入分布式事务、契约版本、链路追踪和部署治理成本。

## 决策

- 需求、场景、执行、评估、缺陷和报告等领域能力初期部署为模块化单体。
- 每个模块拥有明确接口、数据访问边界、测试和负责人。
- 外部系统通过 Ports and Adapters 接入，不直接访问领域表。
- 只有出现独立发布团队、显著扩缩容差异、故障隔离或技术栈需求时才拆分服务。

## 影响

### 正向

- 降低早期交付和运维复杂度。
- 保留清晰边界，为后续拆分提供结构基础。
- 便于跨域事务和本地开发。

### 代价

- 必须通过架构测试和代码评审防止模块边界退化。
- 单个部署单元的发布影响面较大，需要模块级功能开关和回归测试。
