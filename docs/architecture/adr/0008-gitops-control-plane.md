# ADR-0008：应用控制面采用 GitOps 事实源

- 状态：Accepted
- 日期：2026-07-23

## 背景

v0.1 以应用 Manifest 为注册来源，并同步维护应用目录、服务目录和契约锁。v0.2 需要增加版本、兼容、发布、回滚和归档，同时继续支持独立仓库、内网与离线交付。

## 决策

应用声明、不可变版本、兼容策略和环境发布修订以平台 Git 仓库为权威来源。本地 CLI 生成和验证差异，用户通过组织既有 PR/MR 与 CI 流程提交。Control Plane 只读取已验证快照，不允许数据库反写 Git，也不绑定特定 Git 托管 API。

IAM、会话、审计和运行时健康不进入 Git，分别由 PostgreSQL 和可观测基础设施拥有。

## 后果

优点是离线可审查、版本历史天然可追溯、无需在 Control Plane 保存高权限 Git 凭据。代价是应用治理变更具有 Git/CI 延迟，Portal 不能假装同步修改声明；界面必须展示提案、差异和流水线状态。
