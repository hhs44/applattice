# ADR-0002：分离控制面与数据面

- 状态：建议采纳
- 日期：2026-07-17

## 背景

测试平台既处理低容量、高价值的业务命令与元数据，也处理 Camera、LiDAR、Radar、CAN 等大体量连续数据。将两类流量放在相同 API 路径会造成网关拥塞、成本上升和故障放大。

## 决策

- API Gateway / BFF 处理认证、查询、业务命令和数据位置等元数据。
- 原始传感器和大型结果文件通过流式接入、分片上传或对象存储签名地址进入数据面。
- 执行面只回传状态、摘要、数据引用和质量指标。
- 数据面使用独立的限流、分区、重放、质量与生命周期策略。

## 影响

### 正向

- 业务 API 与大数据流量相互隔离。
- 数据处理可以独立扩缩容、回放和补数。
- 安全策略可以按元数据和原始数据分别制定。

### 代价

- 需要维护数据引用一致性和孤儿对象清理。
- 需要统一 `run_id`、数据契约和血缘。
